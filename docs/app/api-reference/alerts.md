# Alerts API

Alerts are system-generated records created by the Sentinel worker pipeline when a detection rule matches an incoming event or when a correlation rule completes. The Alerts API is read-only; you cannot create or modify alerts through the API.

**Base URL:** `http://localhost:4000`

All endpoints require authentication via a session cookie or an `Authorization: Bearer <api-key>` header, and active organisation membership.

---

## Authentication and scopes

| Method | Required role | Required scope |
|--------|--------------|----------------|
| `GET` | viewer, editor, admin | `api:read` |

## Alert object

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string (numeric)` | Auto-incrementing alert ID (backed by a `bigserial` column). Passed as a string to avoid JavaScript integer overflow. |
| `orgId` | `string (UUID)` | Organisation that owns this alert. |
| `detectionId` | `string (UUID) \| null` | Detection that triggered this alert. `null` if the detection was deleted after the alert was created. |
| `ruleId` | `string (UUID) \| null` | Specific rule within the detection that matched. `null` for correlation-triggered alerts. |
| `eventId` | `string (UUID) \| null` | Event that caused the alert to fire. `null` for correlation-triggered alerts. |
| `severity` | `"critical" \| "high" \| "medium" \| "low"` | Copied from the detection or correlation rule at alert creation time. |
| `title` | `string` | Alert title generated by the worker. |
| `description` | `string \| null` | Alert description with contextual detail. |
| `triggerType` | `string` | What triggered the alert. See [Trigger types](#trigger-types). |
| `triggerData` | `object` | Structured data describing the trigger. See [Trigger data structure](#trigger-data-structure). |
| `notificationStatus` | `"pending" \| "sent" \| "partial" \| "failed" \| "no_channels"` | Aggregate delivery status across all channels. |
| `notifications` | `object[]` | Per-channel delivery records. See [Notifications array](#notifications-array). |
| `createdAt` | `string (ISO 8601)` | Timestamp when the alert was created. |

### Trigger types

| Value | Description |
|-------|-------------|
| `rule_match` | A detection rule evaluated to `true` against a single event. |
| `correlated` | A correlation rule completed (sequence, aggregation, or absence). |

### Trigger data structure

The shape of `triggerData` depends on `triggerType`.

**`rule_match` trigger data**

```json
{
  "ruleType": "github.repo_visibility",
  "moduleId": "github",
  "eventType": "github.repository.publicized",
  "matchedFields": {
    "visibility": "public"
  }
}
```

**`correlated` trigger data**

```json
{
  "correlationType": "sequence",
  "correlationKey": {
    "repository.full_name": "my-org/my-repo"
  },
  "windowMinutes": 60,
  "matchedSteps": [
    {
      "stepName": "ProtectionDisabled",
      "eventId": "evt-uuid-...",
      "eventType": "github.branch_protection.disabled",
      "timestamp": 1706265600000,
      "actor": "octocat",
      "fields": { "repository": { "full_name": "my-org/my-repo" } }
    },
    {
      "stepName": "SecretPushed",
      "eventId": "evt-uuid-...",
      "eventType": "github.push.secrets_detected",
      "timestamp": 1706266200000,
      "actor": "octocat",
      "fields": { "repository": { "full_name": "my-org/my-repo" } }
    }
  ],
  "sameActor": true,
  "actors": ["octocat"],
  "timeSpanMinutes": 10.0,
  "modules": ["github"]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `correlationType` | `"sequence" \| "aggregation" \| "absence"` | Type of correlation that fired. |
| `correlationKey` | `object` | Key-value pairs that linked the correlated events. |
| `windowMinutes` | `number` | Detection window used. |
| `matchedSteps` | `object[]` | Steps that matched, in order. |
| `matchedSteps[].stepName` | `string` | Step name from the correlation rule config. |
| `matchedSteps[].eventId` | `string (UUID)` | Event ID that matched this step. |
| `matchedSteps[].eventType` | `string` | Event type of the matched event. |
| `matchedSteps[].timestamp` | `number` | Unix epoch milliseconds when the event occurred. |
| `matchedSteps[].actor` | `string \| null` | Extracted `actor` or `sender` from the event payload. |
| `matchedSteps[].fields` | `object` | Subset of event payload fields captured for cross-step analysis. |
| `sameActor` | `boolean` | `true` if all matched steps share the same actor. |
| `actors` | `string[]` | Deduplicated list of actors across all matched steps. |
| `timeSpanMinutes` | `number` | Time elapsed between first and last matched step. |
| `modules` | `string[]` | Modules involved across matched steps. |

### Notifications array

The `notifications` field records per-channel delivery attempts made by the dispatcher.

```json
[
  {
    "channelId": "chan-uuid-...",
    "channelType": "slack",
    "status": "sent",
    "sentAt": "2026-01-15T10:05:00.000Z"
  },
  {
    "channelId": "chan-uuid-...",
    "channelType": "email",
    "status": "failed",
    "error": "SMTP connection refused"
  }
]
```

### Alert lifecycle

Alerts are immutable records created by the worker. Their lifecycle is as follows:

1. The worker evaluates an event and finds a matching rule or completed correlation sequence.
2. A new alert row is inserted with `notificationStatus: "pending"`.
3. The dispatcher job sends notifications to all associated channels.
4. `notificationStatus` is updated to reflect the final delivery result.

Alerts are never deleted. If a detection is deleted, `detectionId` is set to `null` on existing alerts.

## Endpoints

### List alerts

```
GET /api/alerts
```

Returns a paginated, filtered list of alerts for the authenticated organisation. Results are ordered by `createdAt` descending.

**Query parameters**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `detectionId` | `string (UUID)` | No | Filter by detection ID. |
| `moduleId` | `string` | No | Filter alerts from detections belonging to this module. |
| `severity` | `"critical" \| "high" \| "medium" \| "low"` | No | Filter by severity. |
| `triggerType` | `string` | No | Filter by trigger type (e.g., `rule_match`, `correlated`). |
| `notificationStatus` | `"pending" \| "sent" \| "partial" \| "failed" \| "no_channels"` | No | Filter by notification delivery status. |
| `search` | `string (max 255)` | No | Case-insensitive substring match on `title` or `description`. |
| `from` | `string (ISO 8601 datetime)` | No | Return alerts created at or after this time. |
| `to` | `string (ISO 8601 datetime)` | No | Return alerts created at or before this time. |
| `page` | `integer (≥1)` | No | Page number. Default: `1`. |
| `limit` | `integer (1–100)` | No | Results per page. Default: `20`. |

**Response `200 OK`**

The list response includes a `detectionName` and `moduleId` field resolved via a subquery join.

```json
{
  "data": [
    {
      "id": "1234",
      "severity": "high",
      "title": "Public Repository Created",
      "description": "Repository my-org/my-repo was made public by octocat.",
      "triggerType": "rule_match",
      "notificationStatus": "sent",
      "createdAt": "2026-01-15T10:05:00.000Z",
      "detectionId": "d1e2f3a4-...",
      "ruleId": "rule-uuid-...",
      "eventId": "evt-uuid-...",
      "detectionName": "Public Repository Created",
      "moduleId": "github"
    }
  ],
  "meta": {
    "page": 1,
    "limit": 20,
    "total": 156,
    "totalPages": 8
  }
}
```

**cURL example**

```bash
curl -s \
  -H "Authorization: Bearer sk_..." \
  "http://localhost:4000/api/alerts?severity=high&moduleId=github&from=2026-01-01T00:00:00Z&limit=50"
```

### Get alert

```
GET /api/alerts/:id
```

Returns a single alert with full detail, including the associated event (if available).

**Path parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | `string (numeric)` | Alert ID. Must be a string of digits. |

**Response `200 OK`**

```json
{
  "data": {
    "id": "1234",
    "orgId": "org-uuid-...",
    "detectionId": "d1e2f3a4-...",
    "ruleId": "rule-uuid-...",
    "eventId": "evt-uuid-...",
    "severity": "high",
    "title": "Public Repository Created",
    "description": "Repository my-org/my-repo was made public by octocat.",
    "triggerType": "rule_match",
    "triggerData": {
      "ruleType": "github.repo_visibility",
      "moduleId": "github",
      "eventType": "github.repository.publicized",
      "matchedFields": { "visibility": "public" }
    },
    "notificationStatus": "sent",
    "notifications": [
      {
        "channelId": "chan-uuid-...",
        "channelType": "slack",
        "status": "sent",
        "sentAt": "2026-01-15T10:05:00.000Z"
      }
    ],
    "createdAt": "2026-01-15T10:05:00.000Z",
    "detectionName": "Public Repository Created",
    "event": {
      "id": "evt-uuid-...",
      "orgId": "org-uuid-...",
      "moduleId": "github",
      "eventType": "github.repository.publicized",
      "externalId": "12345678",
      "payload": {
        "repository": { "full_name": "my-org/my-repo", "visibility": "public" },
        "sender": { "login": "octocat" }
      },
      "occurredAt": "2026-01-15T10:04:55.000Z",
      "receivedAt": "2026-01-15T10:04:56.000Z"
    }
  }
}
```

**HTTP status codes**

| Code | Condition |
|------|-----------|
| `200` | Alert found and returned. |
| `400` | `id` is not a numeric string. |
| `404` | Alert not found or belongs to a different organisation. |

**cURL example**

```bash
curl -s \
  -H "Authorization: Bearer sk_..." \
  "http://localhost:4000/api/alerts/1234"
```

### Get alert stats

```
GET /api/alerts/stats
```

Returns dashboard aggregation statistics for the authenticated organisation. This endpoint does not accept query parameters.

**Response `200 OK`**

```json
{
  "total": 3241,
  "today": 14,
  "thisWeek": 87,
  "activeDetections": 23,
  "bySeverity": {
    "critical": 12,
    "high": 98,
    "medium": 245,
    "low": 410
  },
  "recent": [
    {
      "id": "1234",
      "severity": "high",
      "title": "Public Repository Created",
      "createdAt": "2026-01-15T10:05:00.000Z",
      "detectionName": "Public Repository Created"
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `total` | `integer` | Total number of alerts ever created for the organisation. |
| `today` | `integer` | Alerts created since midnight (local server time) today. |
| `thisWeek` | `integer` | Alerts created in the last 7 days. |
| `activeDetections` | `integer` | Number of detections with `status = "active"`. |
| `bySeverity` | `object` | Alert count keyed by severity level. Levels with zero alerts may be absent. |
| `recent` | `object[]` | Up to 10 most recent alerts with `id`, `severity`, `title`, `createdAt`, and `detectionName`. |

> The `stats` path is a static segment and must appear before `/:id` in the URL. The API router handles this correctly.

**HTTP status codes**

| Code | Condition |
|------|-----------|
| `200` | Stats returned. |

**cURL example**

```bash
curl -s \
  -H "Authorization: Bearer sk_..." \
  "http://localhost:4000/api/alerts/stats"
```
